# OpenClaw Integration

OpenClaw uses a **native plugin** (`clawenvkit-eval`) that registers mock service endpoints as agent tools — identical to how real MCP servers work.

## How It Works

```
1. Entrypoint starts mock service (port 9100)
2. Fetches OpenAPI spec → generates /tmp/eval-tools.json with typed parameters
3. OpenClaw gateway starts → loads clawenvkit-eval plugin
   → Plugin reads JSON → registers tools via TypeBox schema → api.registerTool()
4. Agent sees create_task, list_tasks, etc. as native tools
5. Tool execute() makes direct HTTP call to localhost:9100 (bypasses SSRF)
6. Audit log captures all calls → GradingEngine scores
```

## Why Native Plugin?

OpenClaw blocks `localhost` access via SSRF protection. Instead of patching security checks, we register tools through OpenClaw's official plugin API. The agent uses `create_task(title, priority)` the same way it uses `sendSlackMessage` — completely natural.

**41 mock tool names have zero conflicts** with OpenClaw's 51 built-in tools (ours are domain actions like `create_task`, theirs are generic tools like `web_fetch`).

## Plugin Architecture

```
extensions/clawenvkit-eval/
├── openclaw.plugin.json    # Manifest (id, name, configSchema)
├── package.json            # TypeBox dependency + entry point
└── index.ts                # Read eval-tools.json → registerTool()
```

The plugin:

1. Reads `/tmp/eval-tools.json` (generated by entrypoint from OpenAPI spec)
2. For each tool: maps OpenAPI property types to TypeBox schemas
3. Registers via `api.registerTool()` with proper `name`, `label`, `description`, `parameters`, `execute`
4. `execute()` uses `http.request()` to call `localhost:9100` directly — no SSRF involved

## eval-tools.json Format

Generated automatically from the mock service's OpenAPI spec + task.yaml:

```json
[
  {
    "name": "create_task",
    "description": "Create a new task with title, description, priority, due date.",
    "endpoint": "/todo/tasks/create",
    "method": "post",
    "port": 9100,
    "parameters": {
      "title": {"type": "string", "title": "Title"},
      "priority": {"type": "string", "default": "medium", "title": "Priority"},
      "due_date": {"anyOf": [{"type": "string"}, {"type": "null"}], "title": "Due Date"}
    },
    "required": ["title"]
  }
]
```

## Running

```bash
# Build (once)
docker build -f docker/Dockerfile.openclaw -t clawenvkit:openclaw .

# Run any task
docker run --rm \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -v ./dataset/todo/todo-001.yaml:/opt/clawenvkit/task.yaml:ro \
  -v /tmp/results:/logs \
  clawenvkit:openclaw
```

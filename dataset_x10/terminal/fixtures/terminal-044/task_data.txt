# Binary Packet Decoder

## Overview

You are working with a proprietary IoT sensor network that transmits data using a custom binary protocol. Your task is to implement a decoder that can parse these binary packets according to the protocol specification below.

## Protocol Specification

### Packet Structure

Every packet follows this structure:

```
+--------+--------+--------+------------------+----------+--------+
| MAGIC  | VER    | TYPE   | PAYLOAD          | CHECKSUM | END    |
| 2 bytes| 1 byte | 1 byte | variable         | 2 bytes  | 1 byte |
+--------+--------+--------+------------------+----------+--------+
```

**Field Descriptions:**

| Field    | Size     | Description                                      |
|----------|----------|--------------------------------------------------|
| MAGIC    | 2 bytes  | Always `0xAB 0xCD` — identifies a valid packet   |
| VER      | 1 byte   | Protocol version (currently `0x02`)              |
| TYPE     | 1 byte   | Packet type (see table below)                    |
| PAYLOAD  | variable | Type-specific data (see payload formats)         |
| CHECKSUM | 2 bytes  | Sum of all bytes from VER through end of PAYLOAD |
| END      | 1 byte   | Always `0xFF`                                    |

### Packet Types

| Type Code | Name        | Description                        |
|-----------|-------------|------------------------------------|
| `0x01`    | SENSOR_DATA | Environmental sensor readings      |
| `0x02`    | HEARTBEAT   | Device keep-alive signal           |
| `0x03`    | ALERT       | Threshold breach notification      |
| `0x04`    | CONFIG_ACK  | Configuration acknowledgement      |

### Payload Formats

#### SENSOR_DATA (`0x01`)

```
+----------+----------+----------+----------+----------+----------+
| DEVICE_ID| TEMP     | HUMIDITY | PRESSURE | BATTERY  | FLAGS    |
| 4 bytes  | 2 bytes  | 2 bytes  | 2 bytes  | 1 byte   | 1 byte   |
+----------+----------+----------+----------+----------+----------+
```

| Field     | Type             | Encoding                                      |
|-----------|------------------|-----------------------------------------------|
| DEVICE_ID | uint32, big-endian | Raw device identifier                       |
| TEMP      | int16, big-endian  | Temperature in 0.01°C units (e.g. 2315 = 23.15°C) |
| HUMIDITY  | uint16, big-endian | Relative humidity in 0.1% units (e.g. 654 = 65.4%) |
| PRESSURE  | uint16, big-endian | Pressure in 0.1 hPa units (e.g. 10132 = 1013.2 hPa) |
| BATTERY   | uint8              | Battery level 0–100 (percentage)            |
| FLAGS     | uint8              | Bit flags (see below)                       |

**FLAGS Bit Definitions:**

| Bit | Name          | Description                        |
|-----|---------------|------------------------------------|
| 0   | LOW_BATTERY   | Set if battery < 20%               |
| 1   | SENSOR_ERROR  | Set if any sensor reading is invalid |
| 2   | TAMPER        | Set if device tamper detected      |
| 3   | CALIBRATING   | Set if sensor is in calibration mode |
| 4–7 | Reserved      | Must be 0                          |

#### HEARTBEAT (`0x02`)

```
+----------+----------+----------+
| DEVICE_ID| UPTIME   | SEQ_NUM  |
| 4 bytes  | 4 bytes  | 2 bytes  |
+----------+----------+----------+
```

| Field     | Type               | Encoding                          |
|-----------|--------------------|-----------------------------------|
| DEVICE_ID | uint32, big-endian | Raw device identifier             |
| UPTIME    | uint32, big-endian | Seconds since device boot         |
| SEQ_NUM   | uint16, big-endian |

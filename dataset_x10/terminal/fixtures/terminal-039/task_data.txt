# Binary Packet Decoder

## Overview

You are working on a network diagnostics tool that needs to decode binary packets according to a custom protocol specification. Your task is to implement a packet decoder that can parse raw binary data and extract structured information from it.

## Protocol Specification

### Packet Structure

Every packet follows this structure:

```
+------------------+------------------+------------------+------------------+
| HEADER (8 bytes) | TYPE (1 byte)    | LENGTH (2 bytes) | PAYLOAD (N bytes)|
+------------------+------------------+------------------+------------------+
| CHECKSUM (4 bytes)                                                        |
+------------------+------------------+------------------+------------------+
```

### Header Format (8 bytes)

```
Bytes 0-3: Magic number (0xDEADBEEF)
Bytes 4-5: Protocol version (uint16, big-endian)
Bytes 6-7: Sequence number (uint16, big-endian)
```

### Packet Types

| Type Byte | Name        | Description                        |
|-----------|-------------|------------------------------------|
| 0x01      | HEARTBEAT   | Keep-alive signal, no payload      |
| 0x02      | DATA        | Generic data transfer              |
| 0x03      | ACK         | Acknowledgement packet             |
| 0x04      | ERROR       | Error notification                 |
| 0x05      | COMMAND     | Command execution request          |

### Payload Formats

#### HEARTBEAT (0x01)
No payload. Length field must be 0x0000.

#### DATA (0x02)
```
Bytes 0-3:   Timestamp (uint32, big-endian, Unix epoch)
Bytes 4-5:   Channel ID (uint16, big-endian)
Bytes 6-N:   Raw data bytes
```

#### ACK (0x03)
```
Bytes 0-1:   Acknowledged sequence number (uint16, big-endian)
Bytes 2-3:   Status code (uint16, big-endian)
```

#### ERROR (0x04)
```
Bytes 0-1:   Error code (uint16, big-endian)
Bytes 2-N:   Error message (UTF-8 encoded string)
```

#### COMMAND (0x05)
```
Bytes 0-1:   Command ID (uint16, big-endian)
Bytes 2-3:   Flags (uint16, big-endian)
Bytes 4-N:   Command arguments (UTF-8 encoded string)
```

### Checksum

The checksum is a CRC-32 computed over all bytes from the start of the header through the end of the payload. It is stored as a uint32 in big-endian format.

## Your Task

Implement the file `decoder.py` with the following functions:

### `decode_packet(raw_bytes: bytes) -> dict`

Parses a raw binary packet and returns a dictionary with the decoded fields.

**Returns a dictionary with these keys:**

- `"magic"` — integer, the magic number
- `"version"` — integer, protocol version
- `"sequence"` — integer, sequence number
- `"type"` — string, packet type name (e.g., `"HEARTBEAT"`)
- `"type_byte"` — integer, raw type byte value
- `"length"` — integer, payload length in bytes
- `"payload"` — dict, decoded payload fields (empty dict for HEARTBEAT)
- `"checksum_valid"` — boolean, whether the checksum matches
- `"checksum_stored"` — integer, the checksum value stored in the packet
- `"checksum_computed"` — integer, the checksum value computed from the data

**Raises:**

- `ValueError` with message `"Invalid magic number"` if bytes 0-3 are not `0xDEADBEEF`
- `ValueError` with message `"Unknown packet type: 0xXX"` (hex formatted) for unrecognized type bytes
- `ValueError` with message `"Packet too short"` if the byte string is fewer than 15 bytes (minimum valid packet size)

# Binary Packet Decoder

## Background

You are working on a network diagnostics tool that needs to decode binary packets
according to a custom protocol specification used by embedded IoT devices. The
devices transmit sensor data, status updates, and control messages in a compact
binary format to minimize bandwidth usage.

## Protocol Specification

### Packet Structure

Every packet follows this layout:

```
+--------+--------+--------+------------------+----------+
| MAGIC  | TYPE   | FLAGS  | PAYLOAD          | CHECKSUM |
| 2 bytes| 1 byte | 1 byte | variable         | 2 bytes  |
+--------+--------+--------+------------------+----------+
```

**MAGIC**: Always `0xAB 0xCD` — identifies a valid packet start.

**TYPE**: Packet type identifier
- `0x01` — SENSOR_DATA
- `0x02` — STATUS_UPDATE
- `0x03` — CONTROL_MESSAGE
- `0x04` — ERROR_REPORT
- `0xFF` — HEARTBEAT

**FLAGS** (bitmask):
- Bit 0 (`0x01`) — COMPRESSED: payload is zlib-compressed
- Bit 1 (`0x02`) — ENCRYPTED: payload is encrypted (not used in this task)
- Bit 2 (`0x04`) — FRAGMENTED: packet is part of a multi-packet message
- Bit 3 (`0x08`) — PRIORITY: high-priority packet
- Bits 4-7 — reserved, should be 0

**PAYLOAD**: Variable length, structure depends on packet type (see below).

**CHECKSUM**: CRC-16/CCITT-FALSE of all bytes from MAGIC through end of PAYLOAD.

---

### Payload Formats

#### SENSOR_DATA (`0x01`)

```
+----------+----------+--------+--------+--------+--------+
| LENGTH   | SENSOR_ID| COUNT  | READING 1 ...           |
| 2 bytes  | 4 bytes  | 1 byte | COUNT * 9 bytes         |
+----------+----------+--------+--------+--------+--------+
```

Each READING:
```
+----------+----------+----------+
| TIMESTAMP| VALUE    | UNIT     |
| 4 bytes  | 4 bytes  | 1 byte   |
+----------+----------+----------+
```

- LENGTH: total payload length in bytes (big-endian uint16)
- SENSOR_ID: unique sensor identifier (big-endian uint32)
- COUNT: number of readings (uint8)
- TIMESTAMP: Unix timestamp (big-endian uint32)
- VALUE: IEEE 754 float (big-endian)
- UNIT: unit code
  - `0x01` — Celsius
  - `0x02` — Fahrenheit
  - `0x03` — Percent (humidity/battery)
  - `0x04` — Pascal (pressure)
  - `0x05` — m/s (wind speed)
  - `0x06` — Lux (light)
  - `0x07` — ppm (CO2/gas)

#### STATUS_UPDATE (`0x02`)

```
+----------+----------+----------+----------+----------+
| LENGTH   | DEVICE_ID| UPTIME   | BATTERY  | STATUS   |
| 2 bytes  | 4 bytes  | 4 bytes  | 1 byte   | 1 byte   |
+----------+----------+----------+----------+----------+
```

- LENGTH: total payload length (big-endian uint16)
- DEVICE_ID: device identifier (big-endian uint32)
- UPTIME: seconds since boot (big-endian uint32)
- BATTERY: battery percentage (uint8, 0-100)
- STATUS: device status code
  - `0x00` — IDLE
  - `0x01` — ACTIVE
  - `0x02` — SLEEPING
  - `0x03` — ERROR
  - `0x04` — CALIBRATING
